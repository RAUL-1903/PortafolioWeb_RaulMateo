Semanas 10 y 11

APIs Modernas con Spring

Desarrollada del [DIA] al [DIA]

¿Qué temas aprendió?

Di un salto cualitativo hacia la construcción de APIs modernas. Aprendí a utilizar el framework Spring Boot para acelerar el desarrollo, implementando una arquitectura en capas que separa la lógica de negocio de los datos. También dominé la conexión a bases de datos MySQL y la importancia de validar cada "endpoint" utilizando herramientas profesionales como Postman.

Definiciones

Spring Boot

Framework de Java que facilita la creación de aplicaciones independientes y de grado de producción.

MySQL

Sistema de gestión de bases de datos relacional utilizado para persistir la información de la aplicación.

Arquitectura en Capas

Patrón de diseño que separa el código en Controlador, Servicio y Repositorio para mantener el orden.

Postman

Plataforma API para construir y utilizar APIs, utilizada aquí para simular peticiones de cliente.

Procedimientos

  1. Construcción de API: Desarrollo de una API completa desde cero usando Spring Boot.
  2. Simulación de cliente: Uso intensivo de Postman para actuar como cliente y enviar datos de prueba.
  3. Verificación de respuestas: Análisis de códigos de estado HTTP (200 OK, 404 Not Found, 500 Server Error).
  4. Gestión de Dependencias: Inclusión de drivers de MySQL y librerías de Spring.

Resultados

  1. Implementación exitosa de una arquitectura limpia separando lógica y datos.
  2. Conexión estable y funcional entre la aplicación Java y la base de datos MySQL.
  3. Visión clara del flujo de información gracias a las pruebas exhaustivas de endpoints.
  4. Capacidad para identificar y corregir errores basándose en los códigos de respuesta del servidor.
