Novena semana

Backend con Java

¿Qué temas aprendió?

Durante esta semana, comprendí cómo funciona el desarrollo backend a nivel empresarial. Aprendí a configurar servidores web utilizando Apache Tomcat y a estructurar aplicaciones robustas mediante Jakarta EE. Entendí la importancia de los Servlets para procesar peticiones HTTP y cómo se gestiona el ciclo de vida de una aplicación web directamente desde el servidor.

Definiciones

Jakarta EE

Conjunto de especificaciones para el desarrollo de aplicaciones empresariales en Java, incluyendo tecnologías web.

Apache Tomcat

Un contenedor de servlets de código abierto que implementa las especificaciones de Jakarta EE para ejecutar páginas web Java.

Servlet

Clase en Java que extiende las capacidades de un servidor para responder a peticiones, generalmente HTTP.

Archivo .WAR

Web Application Resource. Es un archivo comprimido que contiene todos los recursos de una aplicación web Java lista para desplegar.

Procedimientos

  1. Creación de proyectos dinámicos: Se generaron proyectos Java Web estructurados para soportar tecnologías backend.
  2. Generación de archivos WAR: Empaquetado de la aplicación en archivos .war para su distribución.
  3. Despliegue manual: Carga de la aplicación directamente en el servidor Apache Tomcat.
  4. Configuración de puertos: Ajuste y solución de conflictos en los puertos de conexión del servidor local.

Resultados

  1. Comprensión profunda de la infraestructura que sostiene las aplicaciones web empresariales.
  2. Capacidad para desplegar aplicaciones manualmente sin depender exclusivamente de herramientas automáticas del IDE.
  3. Habilidad para solucionar errores de conexión y conflictos de puertos en el servidor.
  4. Dominio del ciclo de vida de una petición HTTP a través de Servlets.
